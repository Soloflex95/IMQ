# IMQ Core Workflows

## 1. Worker Qualification Workflow

Purpose:
Capture and maintain a worker’s qualifications in a structured, auditable way.

Steps:
1. Worker logs into system
2. Worker enters or updates:
   - Work experience
   - Education
   - Certifications
   - Skills
3. System validates required fields
4. Record is saved as a Qualification Record
5. Audit trail is created

---

## 2. Job Requirement Definition Workflow

Purpose:
Define what is required for a role.

Steps:
1. Administrator or Manager creates Job Title
2. Define Job Requirements:
   - Required skills
   - Required education
   - Required certifications
3. Save Job Requirement
4. Version control / audit record created

---

## 3. Assessment Workflow (Core Engine)

Purpose:
Determine if a worker is qualified for a role.

Steps:
1. Manager assigns Job Requirement to Worker
2. System compares:
   - Worker Qualification Record
   - Job Requirements
3. System generates:
   - Match results
   - Gaps (deficiencies)
4. Manager reviews results
5. Manager selects:
   - Qualified
   - Not Qualified
   - Requires Further Review
6. Manager applies electronic signature
7. Final Assessment is stored
8. Audit record created

---

## 4. Audit / Inspection Workflow

Purpose:
Provide evidence of qualification during audits.

Steps:
1. Auditor requests qualification records
2. Admin selects relevant workers / roles
3. System generates report:
   - Qualification data
   - Assessment results
   - Audit trail
4. Report is published to auditor (controlled access)
5. Access is time-restricted and logged

---

## 5. Periodic Review Workflow

Purpose:
Ensure ongoing compliance over time.

Steps:
1. System triggers periodic review (time-based or event-based)
2. Manager is notified
3. Manager reviews:
   - Worker qualifications
   - Role requirements
4. Manager re-confirms or updates assessment
5. New audit record created

---

## 6. Change Management Workflow

Purpose:
Ensure controlled updates to requirements and qualifications.

Steps:
1. Change is proposed (requirement, workflow, or rule)
2. Change is reviewed
3. Impact is assessed
4. Change is approved or rejected
5. Approved change is implemented
6. Audit record created
